Command Definitions

Types

CommandDefinition

Constructor
CommandDefinition(string commandText, string connectionString, CommandType? commandType = null)
parameters commandText - can be command text or stored procedure. connectionString - ADO.NET connection string to SQL database commandType - System.Data.CommandType, if not supplied BattleAxe will query the database to discover if this is a stored procedure or just text. Subsequent uses of the given command text will be recognized and the call to database to determine CommandType will not be run.
CommandTimeout
is a property that sets the SqlCommand.CommandTimeout.

TextDefinition

TextDefinition inherits from CommandDefinition. It sets the CommandType to Text.

ProcedureDefinition

ProcedureDefinition inherits from CommandDefinition. It sets the CommandType to StoredProcedure.



Extension Methods

Execute(...) includes extensions for strings, CommandDefinition, or objects

Execute(this string commandText, string connectionString)

This extension is a plain CommandText execution of the commandText against the database.

Execute<T>(this string commandText, string connectionString, T parameter = null, CommandType? commandType = null)

'T'
is only a constrained to be a class.
commandText
can be a either CommandType.Text or Procedure.
connectionString
is the ADO.NET ConnectionString to the SQL SERVER.
parameter
its properties are used to set the values of the SqlParameters[].
commandType
If the commandType parameter is not designated BattleAxe will query the db schema to determine if it is a stored procedure.
