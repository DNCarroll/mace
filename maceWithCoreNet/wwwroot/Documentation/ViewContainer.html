ViewContainer

Constructing ViewContainers

ViewContainers should be set up as Singletons (see examples below). mace.ts is convention based unless behavior is overridden. When naming a ViewContainer name the ViewContainer according to Url that it should be handling. For example if the ViewContainer is suppose to handle '/Orders' then the ViewContainer should be named OrdersContainer. There are three helper snipptes for building ViewContainers.

Methods

constructor()

Some customization can be done here including designating the Container as the default Container in the case when a url isn't matched to a ViewContainer (this.IsDefault = true;). Can also tell the container what HTML fragments to get ('Views') and what their HTML Container Element is.

Show

This method starts the process of retrieving the View(s) and getting them into their HTML Container Element. It should not be overridden.

IsUrlPatternMatch
This method determines if the ViewContainer is a match to the current Url. It should not normaly be overridden.
ViewLoadCompleted
This method helps cycle through View(s) to insure smooth load of the HTML fragments to their HTML Containers. It should not be overridden.
Url
Specifies the url to set in the browser based on the parameters. It should not normally be overridden.
DocumentTitle
Value for the browser document title. Can override this is have something specific that is needed for the current Url.
UrlPattern
This returns the UrlBase field if it is not overridden. If Url is not standard then you will want to override this method.
UrlTitle
This value is provided to the History Manager. Can be overridden.

SingleViewContainer

An extension of the ViewContainer that assumes the ViewContainer has only one view and its view fragment is housed in the server path 'Views' and the view will follow the name of the Container.
