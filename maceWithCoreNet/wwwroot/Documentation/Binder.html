Binder Construction

Binders will normally not need to be set up unless customization is required. Binders are attached to HTML View Fragments via the attribute data-binder. They are not connected to Views or ViewContainers directly. Snippets Package contains a snippet for Binders.

Binder Types

Binder is the base class mace.ts. It implements IBinder. Relevant methods on next page. Binders normally have no naming convention connections as they are declared through data-attribute of HTML View Fragments. The exception would be when inheriting and customizing the binder. Binders naming convention in that case default to point to /Api/{BinderName} (Binder keyword is removed when using the by convention method.)

Methods

constructor(...)

parameters: primaryKeys: array of string field that designate the PrimaryKey(s) for incoming DataObjects. Keys are used to find DataObjects within tabular data for returns from POST, PUT, AND DELETE Api Methods. api: If not supplied uses naming convention {Name}Binder to determine the api path. autoUpdate: If set to true then when property change events occur ajax calls are made to the PUT method automatically TypeObject: In the case when the default 'DataObject' of mace.ts is not used. Must implement IObjectState. staticProperties: as of this documentation this feature has not been used. The thought here was the client receives data from a dynamic object from server and properties that should be attached are not. This string array insures that the client has accounted for static property fields that should occur.

ApiPrefix()

Can be overridden to some other path than '/Api/'. If overridden the naming convention for the 'Binder' will still hold true (ApiPrefix() + {BinderName})

RunWhenObjectsChange()
is just a callback method that is hit when the DataObject state changes. This method has no normal functionality only when custimization requires its use and sets it. This accounts for any object or array of objects that are bound to HTML View Fragments.
Add(obj)
can be attached utilized in javascript to add a new object tabular data or form data.
Delete(sender)
can be attached to a button in javascript to delete the row level object from tabular or form data.
Save(obj)
can be attached to click event of HTMLElement to save the associated DataObject to the standard PUT Api method.
SaveDirty()
can be attached to button to save all dirty DataObjects back to an PUT Api method called 'SaveDirty'.

Examples



note - this example shows implementation of custom binder and the RunWhenObjectsChange() method. In this case its to enable disable a save button for Dirty state objects in a tabular view.

Add(obj) - implemented through javascript once an object is formed up the Binder.Add(obj) method can be called. The Add(obj) method points to the Api POST method.

Delete(sender) - The onclick below references a Delete method which is a prototype of a HTMLElement that calls the associated Binder.Delete(sender) method.
<input type="button" value="Delete" onclick="this.Delete()" />

Save(obj) - within the GitHub project from /Views/Order.html. The onclick below references a Save method which is a prototype of a HTMLElement that calls the associated Binder.Save(obj) method.
<input type="button" value="Save" data-class="SaveButtonClass" onclick="this.Save()" />

SaveDirty() - within the GitHub project of this project from /Views/Orders.html. The onclick below references a SaveDirty method which is a prototype of HTMLElement that calls the associated Binder.SaveDirty() method.
<input type="button" id="SaveOrders" onclick="this.SaveDirty()" class="btn btn-success right disabled" style="float:right;" value="Save" />
