The Order 1 or Order #? View is a form view of an order. It calls /Order Api Controller for its Data and Updates. The Save button's class is bound to the data object via a custom object Order that inherits from the DataObject of mace.ts. When the Object has a change the bound class changes appropriately. Upon succes from PUT method for an update the button becomes disabled.

The Orders View is a tabular view of the orders it does not automatically update. Any changes to the array of data from the GET from /Orders Api Controller effect the Save button making it active or inactive based on existence of any records in a 'Dirty' state. Note that this use case assumes that you have an Api PUT Method called SaveDirty on the Api. The Orders View also demonstrates the use of more records functionality.

The AutoUpdate Orders is also a tabular view of the orders but it updates automatically.
