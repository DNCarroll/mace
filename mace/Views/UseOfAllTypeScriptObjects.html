    How to make the grid to left from zero with all the custom classes instantiated. To just do bare bones approach go here
    • Create new project at least 4.5.2.
      To test this implementation I start with an ASP.NET Web Application(.NET Framework) with WebApi (at the time of test Core w/Framework was giving reference exceptions).
    • Set the Web Project specific page to blank.
    • Add folders Api, Scripts, Styles, and Views to the web app.
    • Add Nuget package
      http://www.nuget.org/packages/mace.ts/
    • Add addin Mace.ts.Extensibility from Tools | Extensions and Updates
    • Add a C# file to the project called RouteHandler. Then use the MaceTS extension (just installed) Route Handler for mace.ts Type Project to stub out that file.
    • Add some routes to the Global.asax Application Start up. Can use the MaceTS extension Add Some Routes
    • Create an Controller in the Api folder. I use the Route Attributes so need to make sure have [RoutePrefix("Api/UseOfAllTypeScriptObjects")] and Route... on the methods. Also make sure to have Controller on suffix of the Controller. Web Api Attributes
    • Create a DataObject TypeScript file in Scripts Folder. Use the MaceTS snippet Extending DataObject to create it.
    • Create a Binder TypesScript file in the Scripts folder. The name of this binder should follow naming convention from the WebApi eg WebApi named UseOfAllTypeScriptObjectsController; the Binder should be named UseOfAllTypeScriptObjectsBinder. Use the MaceTS snippet Extending Binder. You need to add to the super constructor the object just created in step above super should look like super(['ID'], null, MaceTestObject);
    • Create a html View in the Views folder. To run the example name it by same convention as above should end up with UseOfAllTypeScriptObjects.html in the Views Folder. Delete everything from the html create. Use MaceTS snippet "Extending View". Change the header data-binder to the Name of the binder from previous step eg data-binder="UseOfAllTypeScriptObjectsBinder". Add any customizations you want for styling to the ul or whatever your repeater is. Note that the repeater is desingated by data-template="row"
      Also note that the alternating class and the more items.
    • Create a View TypeScript file in the Scripts folder. Give it name by same convention as the html view eg UseOfAllTypeScriptObjects.html:UseOfAllTypeScriptObjectsView. Stub this out with the MaceTS snippet "Extending View"
    • Global alternating rows can be achieved by using the Initializer like:
      Initializer.WindowLoaded = (e) => { DataObject.DefaultAlternatingRowClass = "formattedTableHighlight"; }; to do this add a Main.ts to the script and past the code above and set the class with whatever you have made.
    • Create a ViewContainer file in the Scripts folder. Give it a name by same convention as the html view eg UseOfAllTypeScriptObjects.html:UseOfAllTypeScriptObjectsContainer. Stub this out with the MaceTS snippet "Extending a View Container with static View". Make the class name the same as the file name. For the View stub point the view from above. Mark the this.IsDefault = true;
    • Add all the script links to the default.html page. The order is important as the ViewContainer will be referencing the View TS. So the script files for safety can be added in order of Object, Binder, View, ViewContainer. Put mace.ts above them all. Add the Main.ts if it was created. It should be below the mace.ts.
